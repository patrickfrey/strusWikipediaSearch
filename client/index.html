Description of this demo search

  • A documentation of how the search index was configured and built can be found here (building the strus Wikipedia demo search).
  • This is a demo system that has been built to show the performance of strus under conditions of limited resources. Older hardware has been used, because a system pushed to its limits is better to judge. Because the queries will not be very frequent on this system, the chance that the engine has to access disk to retrieve the result, is very high. The size of the index is 6 times bigger than the memory available. Be aware of the size of the index (more than 15 million documents). Normally a search index of this size would be split into smaller units. If somebody thinks strus is slow, so be it. This demo tries to address a public that is aware of what is happening here.

    • Hardware description

    1. Motherboard: Intel S3000AHV
    2. Processor: 1 Intel Xeon Processor 3040, 1.86 GHz
    3. Memory: 8 GB DDR2 RAM, 533 MHz
    4. Hard disc: Western Digial 2TB SATA, product number: WDC WD2002FYPS-0
    5. Index size on disc: ~50G

    Weighting schemas:

    1. BM25:

      Classical BM25 with k1=0.75, b=2.1, avgdoclen=500 plus an extra weight added for link popularity: w=tanh(n/100), where n is the number of links pointing to the page (df of references). Return only documents that contain all query terms.

    2. NBLNK:

      Extract links weighted according their appearance in sentences that contain query terms. The documents inspected are the best 100 documents evaluated with BM25 on documents containing all query terms. The weighting of the links is done by accumulating weights of feature expressions build from pairs of query terms. For a query with 4 terms, there are 14 query expressions built. For example for the query "Roger Federer", the links are weighted as follows:

      1. Links appearing in a sentence where the sequence "Roger Federer" ("Roger" followed immediately by "Federer") occurrs with weight 3.0
      2. Links appearing in a sentence where "Roger" appears near "Federer" ins distance of maximum 5 with weight 2.0
      3. Links appearing in a sentence where "Roger" appears near "Federer" ins distance of maximum 20 with weight 1.5
      Weights of a specific link in a document are multiplied. The resulting weights are accumulated by summation for each link target found. The links with the highest weight are returned as result.
