Running a search engine on the Wikipedia collection (English) on a NUC

Modern machines got damn powerful. We show how you can run a fulltext search engine with sophisticated retrieval schemes without any problem on an Intel NUK (i3). Running means that we can build the search index in a reasonable amount of time (5 hours if we use only one machine to build the index) and serve queries with a response time expected nowadays from a search engine. Numbers about how far the engine can be stressed and how many simultaneus queries can be handled can not be provided yet.
